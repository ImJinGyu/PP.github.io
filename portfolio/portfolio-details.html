Overview

부산온나는 일반 회원, 숙소 업체, 관리자 세 가지 역할을 기준으로 기능을 분리한 숙박 예약 플랫폼입니다. 단순한 화면 구현이 아니라 회원 관리, 예약 흐름, 권한 제어, 문의/리뷰, 이메일 인증, 유효성 검증, 문서화까지 웹 서비스 제작 전 과정을 경험한 프로젝트입니다.

Developer Comment

이 프로젝트는 웹 서비스의 기본기를 한 번에 경험하게 해준 작업이었습니다. 이후 실무에서 ASP.NET MVC, MSSQL, REST API, Vue.js 기반 프로젝트를 수행하면서도, 당시 익힌 권한 분리, 데이터 흐름, 유효성 검증, 문서화 경험이 계속 기반이 되었습니다.

이후 기존 소스를 기반으로 SpringBoot 환경으로 마이그레이션을 진행하고 일부 기능 오류를 수정하여 Cafe24 실서버에 개인적으로 재배포하며 레거시 프로젝트를 최신 환경으로 개선하는 경험도 진행했습니다.

Architecture

Spring MVC + JSP + MyBatis + MySQL 기반의 전형적인 서버 렌더링 구조로 화면, 비즈니스 로직, 데이터 계층을 분리했습니다.

Role-based Access

일반 회원, 숙소 업체, 관리자 권한을 분리하여 접근 가능한 기능과 운영 흐름을 구분했습니다.

Validation

이메일 인증, 입력값 검증, 회원 등급별 기능 제어를 적용하며 서비스 운영에서 필요한 기본 검증 흐름을 구성했습니다.

AJAX & Documentation

AJAX 기반 비동기 요청으로 UX를 개선하고 요구사항 정의서, 회의록, 테이블 명세서, 테스트 케이스를 정리했습니다.

What this project proves

권한, 검증, CRUD, 화면-서버 연동, 문서화까지 웹 서비스 제작 전 과정을 경험하며 이후 실무 역량의 기반을 만든 프로젝트입니다.

Modernization

서비스 중단 이후 기존 소스를 기반으로 SpringBoot 마이그레이션과 실서버 재배포를 직접 진행하며 레거시 개선 경험까지 확장했습니다.
